# Security Policy

Thank you for helping keep SmartTime AI and its users safe.

## Reporting a vulnerability

Please do **not** report security vulnerabilities through public GitHub issues.

Instead:
- use GitHub's private vulnerability reporting if available for this repository
- if private reporting is not available, contact the maintainer privately through GitHub before disclosing details

When reporting, please include:
- a clear description of the issue
- affected area or component
- steps to reproduce
- potential impact
- any proof of concept, if safe to share privately

## Disclosure expectations

Please allow reasonable time for review and remediation before public disclosure.

## Scope

Security-sensitive areas may include:
- authentication and authorization flows
- Firebase/backend access control
- timetable or school data exposure
- import/export handling
- API endpoints and infrastructure configuration
